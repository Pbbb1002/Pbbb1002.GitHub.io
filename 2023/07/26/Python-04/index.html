文件

操作系统中以文件为单位管理磁盘中的数据，一般来说文件可分为文本文件、视频文件、音频文件、图像文件、可执行文件等多种类别

文件的编码

文本文件通过使用编码技术将内容翻译成0和1被计算机识别，从而存储在硬盘中。编码技术就是翻译的规则集合，记录内容和二进制间相互转换的逻辑，不同的编码将内容翻译成二进制的内容也是不同的，UTF-8是目前全球通用的编码格式。

计算机只能识别0和1，所以只有将内容翻译成0和1才能保存在计算机中，同时也需要编码将计算机保存的0和1反向编译回可以识别的内容

文件的操作

文件的操作包括打开、关闭、读、写等。对文件的基本操作大致分为三个步骤：

1.打开文件
2.读写文件
3.关闭文件

注意：可以只打开和关闭文件，而不进行任何读写

文件的读取

open( )：

用于打开函数

语法：open(name,mode,encoding)
name：是要打开的目标文件名的字符串（可以包含文件所在的具体路径）
mode：打开文件的模式（访问模式）：只读、写入、追加等
encoding：编码格式（推荐使用UTF-8）
name和mode使用的是位置传参，而encoding使用的是关键字传参

此时的”f”是open函数的文件对象，对象是python中的一种特殊的数据类型，拥有属性和方法，可以使用对象、属性或对象、方法进行访问。

mode 常用的三种基础访问模式：

模式 描述
r 以只读方式打开文件。文件的指针将会放在文件的开头。这是默认模式
w 打开一个文件只用于写入。如果该文件已存在则打开文件，并从头开始编辑，原有内容将会被删除。如果该文件不存在，则创建新文件
a 打开一个文件用于追加。如果该文件已存在则打开文件，新的内容将会被写入已有内容之后。如果该文件不存在，创建新文件进行写入。

read( )方法：

用于读取指定长度字节
语法：文件对象.read(num)
num 表示要从文件中读取的数据的长度（单位为字节），如果没有传入num，那么就表示读取文件中所有的数据

readlines( )方法：

可以按照行的方式把整个文件中的内容进行一次性读取，并且返回的是一个列表，其中的每一行的数

读取文件时，只要文件打开后，无论调用了什么方法，指针都会指向上一次最后指向的位置，也就是上一个方法所读取的结尾处继续去读

readline( )方法：

可以一次读取一行内容

close( )：

通过close()关闭文件对象，也就是关闭对文件的占用。如果不调用close，同时程序没有停止运行，那么这个文件将一直被python程序占用

with open语法：

通过with open的语句块对文件进行操作可以在操作完成后自动关闭close文件，避免遗忘掉close方法

文件的写入

write( )：

直接调用write时，内容并未真正写入文件，而是积攒在程序的内存中，称之为缓冲区。当调用flush时，内容会真正写入文件。这样做是避免频繁操作硬盘，导致效率下降。

w模式的特点：

文件不存在，会创建新文件；文件存在时，会清空原有内容。

文件的追加

只需将模式替换成a即可，追加写入的方法和w模式一致

文件不存在时，会创建新文件；文件存在时，追加写入文件。

异常的传递性

异常是具有传递性的，当第一个函数中发生异常，并且没有捕获这个异常时，异常会传递到下一个函数，当该函数也没有捕获处理这个异常时，main函数会捕获这个异常。

当所有函数都没有捕获异常的时候，程序就会报错

模块

Python中的模块是一个Python文件，以 .py 结尾，能够定义函数、类和变量，模块中也能包含可执行的代码块。

Python中有很多不同的模块，每一个模块可以帮助我们快速实现一些功能。

模块的导入

模块在使用之前需要先导入，模块的导入一般在代码文件的开头位置
语法：
[ from 模块名 ] import [ 模块 | 类 | 变量 | 函数 | * ] [as 别名 ]
常见的组合形式：
· import 模块名

· from 模块名 import 类、变量、方法等

· from 模块名 import *

· import 模块名 as 别名

· 使用as给特定功能加上别名

自定义模块

当需要一些个性化的模块时，可以通过自定义模块实现，也就是自己制作一个模块

流程：新建一个Python文件 —— 定义函数 —— 调用模块

注意事项：当导入多个模块的时候，且模块内有同名功能，当调用这个同名功能时，调用到的是后面导入的模块的功能

__ main __变量

当开发人员写完一个模块后想要在py文件中添加测试信息时，此时无论是当前文件还是其他已经导入该模块的文件，在运行时都会自动执行测试信息

为解决这个问题，引用 __ main __变量

在模块内部中，**__ name __ 内置变量名将自动标记为__ main __**，从而执行测试代码，在其他文件中测试代码则不被执行

__ all __变量

如果一个模块文件中有__ all __变量时，当使用 from xxx import * 导入时，则只能导入这个列表中的元素

_ _ all _ _ 的定义决定了 * 所代表的’’所有’’的范围，手动导入时不受_ _ all _ _的控制

包

从物理上看，包就是一个文件夹，在该文件夹下包含了一个**__ init __ .py文件，该文件夹可用于包含多个模块文件
从逻辑上看，包的本质依旧是模块**

当我们的模块文件越来越多时，包可以帮助我们管理这些模块名，其作用就是包含多个模块
